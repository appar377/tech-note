## INSERT SELECTの使いどころ

`INSERT SELECT` は、検索結果をそのまま別テーブルへ投入したいときに使うSQLです。アプリケーション側で一度配列に展開せず、データベース内で処理を完結できます。

> [!TIP]
> 大量データを扱う場合は、RubyのループよりもSQL側でまとめて処理した方が速く、メモリ使用量も安定します。

## 基本形

```sql title="insert_select.sql"
INSERT INTO archived_users (user_id, email, archived_at)
SELECT id, email, NOW()
FROM users
WHERE deleted_at IS NOT NULL;
```

| 句 | 役割 |
| --- | --- |
| `INSERT INTO` | 投入先のテーブルとカラム |
| `SELECT` | 投入する値を作る検索 |
| `WHERE` | 対象行の絞り込み |

## ActiveRecordから実行する

```ruby title="archive_users.rb"
sql = <<~SQL.squish
  INSERT INTO archived_users (user_id, email, archived_at)
  SELECT id, email, NOW()
  FROM users
  WHERE deleted_at IS NOT NULL
SQL

ActiveRecord::Base.connection.execute(sql)
```
